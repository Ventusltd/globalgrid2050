# Cable Geometry Visualiser V2 — Architecture & Maintenance Guide

This document outlines the modular architecture of the Cable Geometry Visualiser V2. The application is designed using a strict **Separation of Concerns** to ensure that electrical engineers can safely update cable dimensions without risking the core application logic.

## File Structure

The application is broken down into 6 distinct files:

### 1. The Frontend & Engine
* **`index.html`**: The main user interface. Contains all CSS styling, HTML layout, warning boxes, and the `<canvas>` elements. It loads the database scripts first, followed by the engine.
* **`app.js`**: The core logic engine. Handles all UI event listeners, mathematical geometry calculations, trench sizing, and HTML canvas rendering. **Do not edit this file to add cables.**

### 2. The Configuration
* **`data-core.js`**: Contains the global variables, dropdown menu options, routing logic (`VOLTAGE_CLASSES`), and the arrays of Cross-Sectional Areas (`CSA`) assigned to each voltage class. 

### 3. The Cable Databases
These files contain the actual hardcoded Outer Diameters (OD) and Minimum Bending Radii (MBR). They are strictly formatted JSON-style objects. **Engineers should only edit these files to update the tool.**
* **`data-lv.js`**: Standard 0.6/1kV distribution power cables (e.g., BS 5467 Cu/Al SWA & AWA).
* **`data-mv-hv.js`**: Medium and High Voltage power cables (6kV up to 132kV).
* **`data-solar.js`**: DC string cables, flexible AC/DC screened cables, and rigid Aluminium tubular armoured solar cables.

## How to Update or Add a Cable

To add a new cable size or update an Outer Diameter based on a newly confirmed manufacturer datasheet:

1. **Identify the category:** Open the relevant database file (`data-lv.js`, `data-mv-hv.js`, or `data-solar.js`).
2. **Locate the specific string key:** Keys are formatted logically, e.g., `"3c_cu18_95"` means *3-core, Copper, 18kV (33kV class), 95mm²*.
